How it works

Three steps to structured data

No setup required. Send a file, get structured JSON back.

01
Upload your invoice
Send a multipart POST request with your PDF or image file to /parse. No registration needed for local use.
02
AI extraction
Google Gemini reads the invoice text (with OCR fallback for scanned files) and extracts every GST-relevant field.
03
Receive JSON
Get a clean, typed JSON object — seller, buyer, GSTIN, every line item, CGST/SGST/IGST breakdown, and a confidence score.
Features

Built for production

Everything a developer needs to integrate GST invoice parsing reliably.

📄
PDF & Image Support
Native PDFs, scanned documents, and invoice photos all work. pytesseract OCR kicks in automatically when the PDF has no text layer.
🧾
Complete GST Structure
Seller, buyer, GSTIN, invoice date, all line items with HSN/SAC codes, CGST/SGST/IGST per line, and a full tax summary.
⚡
Batch Processing
Submit multiple PDFs in one request via /parse/bulk. Each file gets its own result; one failure never blocks the rest.
🇮🇳
Hindi Language Support
Pass language=hi to parse Hindi or bilingual invoices. All output is returned in English JSON.
🔒
Secure & Stateless
Files are never written to disk. All processing is in-memory only. No invoice data is retained after the response is sent.
📊
Confidence Scoring
Every response includes a confidence_score from 0 to 1. Below 0.7 signals a low-quality extraction that needs human review.

curl example 
# Single invoice
curl -X POST https://your-api/parse \
  -F "file=@invoice.pdf" \
  -F "language=en"

# Bulk — multiple files
curl -X POST https://your-api/parse/bulk \
  -F "files=@inv1.pdf" \
  -F "files=@inv2.pdf" \
  -F "language=en"
